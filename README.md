# مجلس بلوت

لعبة بلوت سعودية احترافية للويب، بواجهة عربية RTL، غرف جماعية لأربعة لاعبين، وتجربة سريعة ضد ثلاثة لاعبين آليين. جميع الحركات والتحقق من الأوراق والحساب تتم على الخادم لمنع تسريب الأيدي أو قبول حركة غير قانونية.

## ما الذي يميزها؟

- محرك TypeScript مستقل وفق ملف قواعد `ksa_tournament`.
- شراء دورتين: صن، حكم أول، حكم ثانٍ، وأشكل.
- دبل وثري وفور وقهوة، مع المفتوح والمقفل.
- متابعة النوع، القطع، والتعلية القانونية في الحكم.
- سرا، خمسين، مائة متسلسلة، مائة متشابهة، أربعمائة، وبلوت.
- حساب رسمي للصن والحكم والكبوت، والفوز عند 152 بنط.
- غرف خاصة برمز من 6 خانات ورابط قابل للمشاركة.
- جلسة ضيف آمنة في Cookie من نوع HttpOnly واستعادة تلقائية للمقعد.
- مزامنة تكيفية سريعة، منع تكرار الحركة، وتحديث ذري للحالة.
- لعب الورقة بنقرتين متتاليتين من اليد دون زر تأكيد منفصل.
- مركز لمة واضح يربط كل ورقة باسم صاحبها وترتيب رميها، مع مراجعة اللمة السابقة والفائز بها.
- شريط معلومات دائم للمشتري، ونوع اللعب، والمضاعفة، ورقم اللمة، والأكلات، وصاحب الدور.
- تصميم متجاوب للجوال والكمبيوتر والشاشات الأفقية القصيرة، مع اختصارات لوحة المفاتيح ودعم تقليل الحركة.

## البنية

```text
app/                  صفحات اللعبة ومسارات API
components/           الواجهة والطاولة والأوراق
lib/baloot/           المحرك والقواعد والمشاريع واللاعب الآلي
lib/server/           الجلسات ومستودع الغرف الموثوق
db/                   مخطط Drizzle لـ Cloudflare D1
drizzle/              ترحيلات قاعدة البيانات
tests/                اختبارات القواعد والرندر
worker/               نقطة تشغيل Cloudflare Worker
```

الحالة الكاملة محفوظة في D1 داخل `rooms.state_json`. كل عميل يحصل على إسقاط خاص بمقعده: يده فقط، أعداد أوراق الآخرين، والحركات القانونية. تحديثات اللعب تستخدم مقارنة `gameVersion` لمنع السباقات، ومعرّف حركة يجعل إعادة الطلب آمنة.

## التشغيل المحلي

المتطلبات: Node.js `>=22.13.0`.

```bash
npm install
npm run db:generate
npm run dev
```

للتحقق:

```bash
npm test
npm run build
npm run test:render
```

## قواعد اللعب

المصدر الأعلى للمحرك هو [اللائحة العامة الرسمية للبلوت](https://enjoy.sa/media/j1ofreng/baloot.pdf). استُكملت تفاصيل رمي الورق غير المفصلة في اللائحة من [Pagat](https://www.pagat.com/jass/baloot.html)، مع مراجعة شرح [جواكر](https://blog.jawaker.com/baloot-rules/).

أمثلة قبول يغطيها الاختبار:

- صن 65–65: تعادل صحيح، 13 لكل فريق.
- وصول خصم المشتري إلى 66 في الصن: سقوط المشتري.
- حكم 81–81: تعادل صحيح، 8 لكل فريق.
- وصول خصم المشتري إلى 82 في الحكم: سقوط المشتري.
- الكبوت: 44 صن و25 حكم حتى في اليد المدبلة وفق اللائحة الرسمية.

## الأمان والخصوصية

- الخادم يخلط ويوزع ويحدد قانونية كل طلب.
- رموز الجلسات لا تحفظ في قاعدة البيانات إلا بعد SHA-256.
- Cookies: `HttpOnly`, `SameSite=Lax`, و`Secure` على HTTPS.
- لا تحتوي اللعبة على حسابات أو بريد أو بيانات دفع.
- لا ترسل API أوراق الخصم أو رزمة التوزيع للمتصفح.

## النشر

المشروع مهيأ لـ OpenAI Sites / Cloudflare Workers، ويعلن ربط D1 باسم `DB` في `.openai/hosting.json`. يحتفظ النشر بالترحيلات داخل `drizzle/` ويطبقها على قاعدة المنصة.
